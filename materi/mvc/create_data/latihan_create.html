Studi Kasus Sesi 13: Pencatatan Barang Masuk

Klien: Gudang "Toko Online"
Masalah: Admin gudang masih mencatat barang baru di buku tulis.
Tugas Anda: Buatkan halaman formulir digital agar admin bisa menginput data produk langsung ke sistem.

Langkah 1: Menyiapkan Jalur (Route & Controller)

Sebelum membuat tampilan, pastikan jalurnya sudah ada. Admin ingin mengakses form ini lewat alamat /products/create.

To-Do List:

  • Buka ProductController.php, tambahkan method create() yang mengembalikan view products.create.
  • Buka web.php, daftarkan route GET untuk alamat tersebut.
  • Cek: Buka browser. Jika muncul error "View not found", berarti jalur sudah benar (karena view belum dibuat).

Langkah 2: Membangun Antarmuka (View)

Buat file resources/views/products/create.blade.php. Desain formulir yang rapi agar admin senang menggunakannya.

Spesifikasi Formulir:

  • Action: Arahkan ke /products (Method POST).
  • Security: Jangan lupa token @csrf.
  • Input Fields:
    • name (Teks)
    • price (Angka)
    • stock (Angka)
    • description (Textarea - Opsional)

Tips: Gunakan class Tailwind seperti w-full, border, p-2, rounded agar tampilannya modern.

Langkah 3: Proses Penyimpanan (Controller)

Form sudah ada, tapi belum bisa menyimpan. Tangani data kiriman admin di method store().

1. Validasi Ketat:

Pastikan Nama Barang & Harga wajib diisi. Stok harus angka.

2. Simpan Data:

Gunakan Product::create($request->all());.
(Pastikan $fillable di Model sudah diatur!)

3. Feedback:

Setelah simpan, jangan biarkan layar putih. Redirect admin kembali ke halaman Daftar Produk (/products) dengan pesan sukses.

Uji Coba Mandiri (Quality Control)

Lakukan skenario berikut untuk memastikan fitur Anda 100% bekerja:

  • Buka halaman form, tekan tombol "Simpan" tanpa mengisi apapun. (Harus muncul pesan error / kembali ke form).
  • Isi data dengan benar: "Kopi Robusta", Harga 15000, Stok 100. Tekan Simpan.
  • Cek apakah halaman berpindah ke Index?
  • Cek apakah "Kopi Robusta" muncul di daftar paling atas?
