Konsep MVC

Model - View - Controller

Arsitektur di balik Laravel

Apa itu MVC?

MVC adalah pola desain perangkat lunak yang memisahkan aplikasi menjadi tiga bagian utama: Data (Model), Tampilan (View), dan Logika (Controller).

ðŸ’¡ Tujuannya: Agar kode rapi, terstruktur, dan mudah dikelola bersama tim.

1. Analogi Restoran

Route

Resepsionis

Menerima tamu (User) di depan pintu dan membaca buku menu (URL) untuk mengarahkan tamu.

Controller

Pelayan (Waiter)

Menerima pesanan, berteriak ke dapur (Model), dan mengantar makanan ke meja (View).

Model

Koki (Chef)

Bekerja di dapur (Database). Mengolah bahan baku (Data) menjadi masakan siap saji.

View

Piring Saji

Tampilan akhir yang cantik (HTML/CSS) yang disajikan dan dilihat oleh tamu.

2. Alur Kerja (Lifecycle)

1

User Request

User mengetik URL di browser. Contoh: website.com/profil

2

Route

web.php

"Ada tamu minta halaman Profil. Panggil ProfilController!"

3

Controller

ProfilController.php

"Siap! Saya butuh data Nama & Foto User. Hai Model User, ambilkan datanya!"

4

Model

User.php

Mengambil data dari Database SQL -> Mengembalikannya ke Controller.

5

View

profil.blade.php

Controller menyerahkan data ke View. View menyusun HTML cantik dan mengirimkannya kembali ke Browser User.

3. Kenapa Harus Pakai MVC?

Terorganisir

Kode tidak menumpuk di satu file. Logic, Database, dan Tampilan punya rumah masing-masing.

Kerja Tim

Si A bisa fokus koding Database (Backend), si B bisa fokus menghias Tampilan (Frontend) tanpa saling ganggu.

Mudah Diperbaiki

Kalau tampilan error, cukup cek View. Kalau perhitungan salah, cukup cek Controller.
