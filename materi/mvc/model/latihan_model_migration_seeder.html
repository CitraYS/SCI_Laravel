1 Definisi & Konsep Dasar

Migration

"Blueprint / Arsitektur".
Desain tabel database menggunakan kode PHP. Jika laptop rusak, struktur database tetap aman di file migration.

Model (Eloquent)

"Manajer Gudang".
Perwakilan tabel di kode program. Kita menyuruh Model untuk mengambil/simpan data, bukan pakai SQL manual.

Latihan Praktik: Tantangan 3 Tabel

Misi: Membangun sistem "Produk & Ulasan" yang utuh.
Siswa diminta membuat 3 tabel yang saling berelasi agar saat dimigrate TIDAK ERROR.

1. Tabel Categories
  • id (PK)
  • name (string)
  • slug (string)

PENTING: Harus dibuat LEBIH DULU daripada products!

2. Tabel Products
  • id (PK)
  • category_id (FK)
  • name, price, stock
3. Tabel Reviews
  • id (PK)
  • product_id (FK)
  • rating (float 1-5)
  • comment (text)

Relasi ke tabel Products

Soal Latihan:

1

Fix Migration Error (Urutan Tabel)

Jika kita jalankan php artisan migrate sekarang, akan error. Kenapa?
Karena tabel products meminta relasi ke categories, tapi tabel categories belum ada.
Tugas: Buat Model & Migration untuk Category. Pastikan file migration category urutannya sebelum file migration products (Tips: Rename tanggal filenya jika perlu).

2

Buat Fitur Review

Buat Model & Migration baru bernama Review.
Tabel ini harus punya kolom: rating (pakai float/decimal) dan comment (text).
Jangan lupa tambahkan $table->foreignId('product_id')->constrained() agar terhubung ke produk.

3

Seeding Relasi

Buat ReviewSeeder. Isi data dummy ulasan untuk produk yang sudah dibuat sebelumnya.
Pastikan product_id yang diinput di seeder BENAR-BENAR ADA di database (misal: id 1, 2, atau 3).
