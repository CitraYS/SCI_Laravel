Bagian 1: The Restaurant Analogy

Pasangkan istilah teknis Laravel di kiri dengan peran di Restoran di kanan.

A. Route
B. Controller
C. Model
D. View
1. Koki / Gudang Bahan
2. Piring Saji
3. Resepsionis
4. Pelayan / Waiter

Kunci Jawaban:

  • A - 3 (Route = Resepsionis)
  • B - 4 (Controller = Pelayan)
  • C - 1 (Model = Koki)
  • D - 2 (View = Piring Saji)

Bagian 2: Detektif Alur

Seorang User ingin membuka halaman "Profil Saya". Urutkan langkah-langkah acak berikut (1-5).

A

Controller meminta data "Nama & Foto" kepada Model.

B

View menampilkan halaman HTML lengkap ke layar User.

C

User mengetik website.com/profil di browser.

D

Model mengambil data dari Database dan memberikannya ke Controller.

E

Route mengecek URL /profil dan memanggil ProfilController.

Urutan Benar:

  1. User mengetik URL
  2. Route memanggil Controller
  3. Controller minta data ke Model
  4. Model kembalikan data ke Controller
  5. View tampilkan ke layar

Bagian 3: Siapa Saya? (Tebak Peran)

"Tugas saya isinya cuma HTML dan CSS. Saya tidak boleh mikir berat-berat."

View

"Saya adalah gerbang pertama. Kalau alamat URL tidak terdaftar, saya kasih Error 404."

Route

"Saya yang punya akses kunci ke Database. Kalau butuh data user, tanya saya."

Model

"Saya otaknya. Saya yang memutuskan data apa yang diambil dan tampilan mana yang dipakai."

Controller

Bagian 4: Studi Kasus (Instagram)

1. Tombol "Like" berbentuk hati berwarna merah.

Apakah ini urusan Model atau View?

Jawaban: View

2. Menyimpan jumlah Likes yang bertambah ke database.

Siapa yang melakukan penyimpanan ini?

Jawaban: Model

3. Logika: "Jika user belum login, jangan tampilkan tombol Like".

Siapa yang mengatur logika pengecekan ini?

Jawaban: Controller
