0. TABLE OPERATIONS (Kelola Wadah)

DDL

Sebelum mengisi data, kita harus membuat wadahnya (Tabel) terlebih dahulu. Ini yang dilakukan Laravel Migration di belakang layar.

A. Membuat Tabel Baru (CREATE)

CREATE TABLE students ( id BIGINT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(255) NOT NULL, email VARCHAR(255) UNIQUE, score INT DEFAULT 0 );
Logika: Buat tabel 'students' dengan kolom id (otomatis urut), nama (wajib isi), email (tidak boleh kembar), dan nilai (default 0).

B. Menghapus Tabel (DROP)

DROP TABLE students;

Create

Read

Digunakan untuk mengambil data. Kita bisa ambil semua atau filter tertentu.

SELECT * FROM students WHERE score > 80;

Update
UPDATE students
SET score = 90
WHERE id = 1;

4. DELETE (Menghapus Data)

Delete
DELETE FROM students WHERE id = 1;

6. JOIN (Mengambil Data Relasi)

Read Join

Bagaimana cara menampilkan Nama Siswa beserta Nama Kelasnya sekaligus? Jika kita hanya melakukan SELECT biasa, kita hanya melihat ID kelas saja. Gunakan JOIN untuk menggabungkannya.

SQL Query:

SELECT students.name, classrooms.name AS class_name
FROM students
JOIN classrooms ON students.classroom_id = classrooms.id;
Cara bacanya: "Ambil nama siswa dan nama kelas (beri nama lain 'class_name'), dari tabel siswa, gabungkan dengan tabel kelas dimana ID kelas di tabel siswa SAMA DENGAN ID asli kelas."
Di Laravel nanti: Jauh lebih mudah! Cukup panggil relasinya:
$student->classroom->name;

7. MULTI JOIN (Gabungan 3 Tabel)

Complex Read

Tantangan: Bagaimana jika kita ingin melihat Nama Siswa, Nama Kelasnya, DAN Nama Wali Kelasnya?
Asumsi: Tabel classrooms punya kolom teacher_id yang terhubung ke tabel baru bernama teachers.

Students ‚ûú Classrooms ‚ûú Teachers

SQL Query:

SELECT
  students.name AS siswa,
  classrooms.name AS kelas,
  teachers.name AS wali_kelas
FROM students
JOIN classrooms ON students.classroom_id = classrooms.id
JOIN teachers ON classrooms.teacher_id = teachers.id;
Di Laravel nanti: Kamu cukup menyambung panahnya (Chaining):
$student->classroom->teacher->name;
