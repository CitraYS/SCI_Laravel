Sesi 13: Input Data (Create)

Forms & Logic

1. Alur Kerja "Create"

Untuk membuat fitur tambah data, kita butuh 2 Rute (Routes):

  1. GET (create): Hanya menampilkan Formulir kosong kepada user.
  2. POST (store): Menerima data dari formulir, memvalidasi, dan menyimpannya ke Database.

2. Logic Controller

Tambahkan 2 method ini di ProductController.php

// 1. Method GET: Hanya untuk MENAMPILKAN Form HTML
// Standar Laravel menamakannya 'create', tapi bisa diganti jadi 'pageCreate' atau 'viewCreate'
public function create()
{
    return view('products.create');
}

Info: Kenapa namanya 'create' dan 'store'?

Ini adalah standar industri (Resource Controller).
- create() = Menyiapkan/menampilkan halaman pembuatan.
- store() = Menyimpan data kiriman user.
Anda boleh menggantinya (misal: pageCreate), tapi Route-nya juga harus diupdate:
Route::get('/products/create', [ProductController::class, 'pageCreate']); 

// 2. Method POST: Menerima & Memproses Data (Simpan)
public function store(Request $request)
{
    // A. Validasi (Security Guard)
    // Pastikan data sesuai aturan sebelum masuk database
    $request->validate([
        'name' => 'required|min:3',
        'price' => 'required|numeric',
        'stock' => 'required|integer',
    ]);

    // B. Simpan ke Database (Cara Cepat / Mass Assignment)
    // Product::create otomatis mencocokkan nama input form dengan nama kolom tabel
    Product::create($request->all());

    // C. Redirect (Kembalikan ke halaman index)
    return redirect('/products')->with('success', 'Produk berhasil ditambahkan!');
}

3. Variasi Simpan Data (Enrichment)

Cara ::create($request->all()) adalah cara "Magic". Namun, kadang kita butuh cara manual.

Cara A: Mass Assignment (Magic)

Paling cepat. Cocok jika nama input form SAMA PERSIS dengan nama kolom database.

Product::create($request->all());

Wajib setting $fillable di Model.

Cara B: Manual OOP (Explicit)

Lebih verbose/panjang. Cocok jika data perlu diubah dulu sebelum disimpan (contoh: upload file).

$product = new Product();
$product->name = $request->name;
$product->price = $request->price + 5000; // Manipulasi
$product->save();

Cara C: Logic di Model (Advanced / Cleaner Controller) Best Practice

Jika logika penyimpanan rumit, Controller cukup "menyuruh" Model. Controller tetap bersih, Model yang bekerja keras (Fat Model, Skinny Controller).

1. Buat Fungsi di Model (Product.php)

// Static function agar bisa dipanggil tanpa 'new'
public static function simpanData($data)
{
    // Manipulasi data di sini (misal: hitung pajak)
    $data['price'] = $data['price'] + 1000;
    
    // Simpan
    return self::create($data);
}

2. Panggil di Controller (ProductController.php)

public function store(Request $request)
{
    // Controller tinggal panggil 1 baris saja
    Product::simpanData($request->all());

    return redirect('/products');
}
Wawasan Industri: Cara C di atas adalah langkah awal menuju arsitektur yang rapi. Untuk aplikasi besar, programmer senior biasanya memindahkan logika tersebut lebih jauh lagi ke dalam class khusus bernama Service atau Repository.

4. Daftarkan Route

File: routes/web.php

PENTING: Route create harus ditaruh DI ATAS route show/{id}. Jika tidak, Laravel akan mengira kata "create" adalah ID produk. 
Route::get('/products/create', [ProductController::class, 'create']); // Tampil Form
Route::post('/products', [ProductController::class, 'store']); // Proses Simpan

5. Membuat Form (Blade)

File: resources/views/products/create.blade.php

Poin Penting Form Laravel:

  • action: Arahkan ke route POST.
  • @csrf: (Wajib) Token keamanan untuk mencegah serangan Cross-Site Request Forgery. Tanpa ini, form akan Expired (Error 419).
  • name="...": Harus SAMA PERSIS dengan nama kolom di database.
<form action="/products" method="POST">
    @csrf <!-- WAJIB ADA -->

    <div class="mb-3">
        <label>Nama Produk</label>
        <input type="text" name="name" class="border p-2 w-full">
        <!-- Pesan Error Validasi -->
        @error('name')
            <p class="text-red-500 text-sm">{{ $message }}</p>
        @enderror
    </div>

    <div class="mb-3">
        <label>Harga</label>
        <input type="number" name="price" class="border p-2 w-full">
    </div>

    <button type="submit" class="bg-blue-500 text-white p-2 rounded">Simpan</button>
</form>
