Proyek Database Toserba "Maju Jaya"

Implementasi Relasi Database untuk Operasional Toko Retail

Deskripsi Tugas

Keluarga temanmu memiliki sebuah Toserba. Mereka kesulitan mencatat stok barang dan transaksi penjualan secara manual. Tugasmu adalah merancang database sederhana untuk mencatat Barang dan Transaksi Penjualan.

Tujuan Akhir:

Kamu harus bisa menampilkan struk belanja lengkap (Nama Barang, Harga, Jumlah Beli) menggunakan Query SQL.

1. Desain Tabel (Schema)

Wajib Dibuat

A. Tabel: categories

Untuk mengelompokkan barang (Makanan, Minuman, Alat Tulis).

  • - id (PK, BigInt, Auto Increment)
  • - name (Varchar 100)

B. Tabel: products

Daftar barang yang dijual.

  • - id (PK, BigInt, Auto Increment)
  • - name (Varchar 255)
  • - price (Int)
  • - stock (Int)
  • - category_id (FK -> categories)

C. Tabel: transactions

Header nota/struk (Siapa yang beli & kapan).

  • - id (PK, BigInt, Auto Increment)
  • - transaction_date (Date/Datetime)
  • - customer_name (Varchar 100)

D. Tabel: transaction_details

Isi keranjang belanja (Pivot Table).

  • - id (PK, BigInt, Auto Increment)
  • - transaction_id (FK -> transactions)
  • - product_id (FK -> products)
  • - quantity (Int) // Jumlah beli

2. Isi Data (Seeding)

Jalankan perintah INSERT ini agar kita punya bahan untuk latihan query.

1. Isi Kategori

INSERT INTO categories (name) VALUES ('Makanan'), ('Minuman'), ('Sabun');

2. Isi Produk

INSERT INTO products (name, price, stock, category_id) VALUES ('Indomie Goreng', 3500, 50, 1), ('Sari Roti', 15000, 10, 1), ('Teh Botol', 5000, 24, 2), ('Aqua 600ml', 3000, 100, 2), ('Lifebuoy Cair', 25000, 5, 3);

3. Buat 2 Transaksi (Budi & Ani)

INSERT INTO transactions (transaction_date, customer_name) VALUES ('2023-10-01', 'Budi'), ('2023-10-02', 'Ani');

4. Isi Keranjang Belanja

-- Budi beli Indomie (5 bks) dan Teh Botol (2 btl) INSERT INTO transaction_details (transaction_id, product_id, quantity) VALUES (1, 1, 5), (1, 3, 2); -- Ani beli Sari Roti (1 bks) dan Lifebuoy (1 pcs) INSERT INTO transaction_details (transaction_id, product_id, quantity) VALUES (2, 2, 1), (2, 5, 1);

3. Tantangan Query (The Exam)

Jawablah kasus operasional berikut dengan SQL.

Level 1: Cek Stok Menipis

Pemilik toko ingin tahu barang apa saja yang stoknya di bawah 15 pcs agar bisa segera kulakan.

Hint: Gunakan SELECT sederhana dengan WHERE operator pembanding.
-- Tulis jawabanmu di sini SELECT name, stock FROM products WHERE stock < 15;

Level 2: Daftar Barang & Kategorinya

Tampilkan nama barang beserta nama kategorinya. Jangan tampilkan ID kategori.

Hint: JOIN tabel products dan categories.
-- Tulis jawabanmu di sini SELECT products.name, categories.name as kategori FROM products JOIN categories ON products.category_id = categories.id;

Level 3: Cetak Struk Belanja (Multi Join)

Tampilkan detail belanjaan milik 'Budi'. Data yang diminta: Nama Pembeli, Tanggal, Nama Barang yang dibeli, Harga Barang, dan Jumlah Beli.

Hint: Kamu butuh 3 tabel. Transactions -> Transaction Details -> Products. Gunakan WHERE customer_name = 'Budi'.
-- Tulis jawabanmu di sini SELECT t.customer_name, t.transaction_date, p.name AS nama_barang, p.price, td.quantity FROM transactions t JOIN transaction_details td ON t.id = td.transaction_id JOIN products p ON td.product_id = p.id WHERE t.customer_name = 'Budi';

Level 4: Laporan Omset (Analisa)

Hitung total belanjaan (Subtotal) untuk setiap item yang dibeli, dan tampilkan hanya yang subtotalnya di atas Rp 20.000.

Hint: Subtotal = price * quantity. Gunakan perkalian di dalam SELECT, lalu filter hasilnya.
-- Tulis jawabanmu di sini SELECT p.name, td.quantity, (p.price * td.quantity) as subtotal FROM transaction_details td JOIN products p ON td.product_id = p.id WHERE (p.price * td.quantity) > 20000;
